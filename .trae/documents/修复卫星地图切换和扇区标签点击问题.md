## 问题分析

1. **卫星地图切换无反应**：

   * 当用户点击卫星地图选项时，mapType状态更新了，但瓦片图层没有相应更新

   * 原因：地图初始化时mapLayerType可能已变为'satellite'，但此时useEffect的依赖项没有包含地图初始化状态，导致瓦片图层未更新

2. **扇区标签点击无反应**：

   * 点击扇区标签开关时，状态没有正确更新，扇区标签未显示

   * 原因：需要检查事件处理和状态更新逻辑

## 修复方案

### 1. 修复卫星地图切换问题

**文件**：`frontend/src/renderer/components/Map/OnlineMap.tsx`

* 添加地图初始化状态跟踪

* 修改瓦片图层更新的useEffect，确保地图初始化后能根据当前mapLayerType更新瓦片图层

```ts
// 添加地图初始化状态
const [isMapInitialized, setIsMapInitialized] = useState(false)

// 在地图初始化完成后设置isMapInitialized为true
// 在初始地图初始化useEffect的最后添加：setIsMapInitialized(true)

// 修改瓦片图层更新的useEffect
useEffect(() => {
  if (!isMapInitialized || !mapInstanceRef.current || !window.L) return
  
  // 瓦片图层更新逻辑...
}, [isMapInitialized, mapLayerType])
```

### 2. 修复扇区标签点击问题

**文件**：`frontend/src/renderer/components/Map/LayerControl.tsx`

* 简化扇区图层的树结构，将扇区可见性和标签显示开关直接作为顶层选项，避免嵌套过深

* 确保事件能正确传递和处理

**文件**：`frontend/src/renderer/components/Map/OnlineMap.tsx`

* 确保labelVisibility变化时能触发扇区图层重建

* 验证createSectorLayers函数的依赖项是否包含labelVisibility

### 3. 优化事件处理

**文件**：`frontend/src/renderer/components/Map/LayerControl.tsx`

* 确保所有点击事件都能正确触发相应的回调函数

* 检查事件冒泡和阻止冒泡的使用是否正确
