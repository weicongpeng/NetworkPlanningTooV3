## 修复方案

### 问题分析
应用到工参后，旧工参被标记为"full_params_backup"但数据仍存在，导致前端同步服务可能获取到错误的数据。

### 修复方案（最高效方法）

在后端 `apply_pci_to_params` 函数中，完成以下操作：

1. **删除所有旧的备份工参**：在创建新工参前，删除所有 `fileType` 为 `full_params_backup` 的数据

2. **删除旧的全量工参**：将当前 `full_params` 标记为备份后，直接删除其数据目录

3. **确保新工参正确注册**：新工参注册为 `full_params`，无需保留旧数据

### 修改文件

**backend/app/api/v1/endpoints/pci.py**：

```python
# 在创建新数据目录之前，删除所有旧的备份工参
for old_id, old_info in list(data_service.index.items()):
    if old_info.get("fileType") == "full_params_backup":
        data_service.delete_data(old_id)
        logger.info(f"[应用PCI到工参] 已删除旧备份工参: {old_id}")

# ... 创建新工参 ...

# 将旧工参标记为备份后，直接删除（而不是保留）
if full_params_id in data_service.index:
    data_service.delete_data(full_params_id)
    logger.info(f"[应用PCI到工参] 已删除旧工参: {full_params_id}")
```

### 优点
- 最高效：直接在后端处理，无需前端额外操作
- 数据干净：不保留任何旧数据，避免混淆
- 自动化：用户无需手动清理
