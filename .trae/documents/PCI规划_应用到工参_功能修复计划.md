# PCI规划"应用到工参"功能修复计划

## 问题分析

通过代码分析，我发现PCI规划中的"应用到工参"功能存在以下问题：

1. **后端功能已实现**：后端API端点`/apply-to-params/{task_id}`已完整实现，可将PCI规划结果更新到全量工参文件（包括data.json和原始Excel文件）

2. **前端调用逻辑存在缺陷**：
   - 前端`PCIPage.tsx`中的`handleApplyToParams`函数已实现，但在调用后端API后，没有确保扇区图层使用更新后的PCI值
   - 扇区图层使用的是传入的`pciData`，而不是从同步服务获取的更新后的数据

3. **数据同步链路不完整**：
   - `pciDataSyncService`负责在内存中同步PCI值，但扇区图层没有直接使用该服务的同步数据
   - 调用"应用到工参"后，地图数据刷新逻辑可能不完整

## 修复方案

### 1. 优化前端`handleApplyToParams`函数

**修改文件**：`frontend/src/renderer/pages/PCIPage.tsx`

**具体修改**：
- 在调用后端API成功后，确保重新获取最新的全量工参数据
- 更新`currentTaskResult`中的PCI值，确保扇区图层使用最新数据
- 增强错误处理和用户反馈

### 2. 改进扇区图层数据使用方式

**修改文件**：`frontend/src/renderer/components/Map/OnlineMap.tsx` 或 `frontend/src/renderer/components/Map/SectorRendererSVG.tsx`

**具体修改**：
- 确保扇区图层在渲染时优先使用`pciDataSyncService`中的同步PCI值
- 添加机制，当PCI值更新时自动刷新图层显示

### 3. 完善数据同步服务

**修改文件**：`frontend/src/renderer/services/pciDataSyncService.ts`

**具体修改**：
- 增强同步服务的可靠性，确保PCI值正确同步到所有相关扇区
- 添加方法，用于获取更新后的完整PCI数据，供扇区图层使用

### 4. 测试验证

**验证步骤**：
1. 执行PCI规划，生成规划结果
2. 点击"应用到工参"按钮
3. 验证后端是否成功更新全量工参文件
4. 验证前端扇区图层是否显示更新后的PCI值
5. 验证"应用到工参"操作是否正确处理错误情况

## 预期结果

1. **功能完整**：PCI规划完成后，点击"应用到工参"按钮，待规划小区的PCI值应成功更新到当前使用的全量工参文件

2. **数据同步**：相应网络类型的扇区图层中的PCI属性应同步显示最新值

3. **用户体验**：操作过程应提供清晰的成功/失败反馈，确保用户了解操作结果

4. **可靠性**：功能应能处理各种边界情况，如工参文件不存在、规划结果为空等

## 技术要点

- **数据一致性**：确保后端文件更新与前端内存数据同步保持一致
- **实时反馈**：操作过程中提供适当的加载状态和结果反馈
- **错误处理**：完善错误捕获和用户提示机制
- **代码可维护性**：确保修改后的代码结构清晰，易于理解和维护
