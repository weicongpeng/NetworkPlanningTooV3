## 代码逻辑对比

### 瓦片加载失败检测

#### 优化前
```typescript
let failedTileCount = 0
const ERROR_THRESHOLD = 3  // 需要 3 个瓦片失败

tileLayer.on('tileerror', (error) => {
  failedTileCount++
  if (failedTileCount >= ERROR_THRESHOLD) {
    // 显示提示
  }
  // 防抖 2 秒
  setTimeout(() => {
    if (failedTileCount > 0) {
      // 显示提示
    }
  }, 2000)
})
```

#### 优化后
```typescript
let hasShownError = false  // 只显示一次

tileLayer.on('tileerror', (error) => {
  // 第一个瓦片失败就立即提示
  if (!hasShownError) {
    hasShownError = true
    setNetworkError({ visible: true, ... })
  }
})
```

### 网络恢复检测

#### 优化前
```typescript
// 只有等待所有瓦片加载完成
tileLayer.on('load', () => {
  // 清除提示（需要等待很久）
})
```

#### 优化后
```typescript
// 方式 1: 浏览器网络状态监听（最快）
window.addEventListener('online', () => {
  setNetworkError(prev => ({ ...prev, visible: false }))
})

// 方式 2: 单个瓦片加载成功（次快）
tileLayer.on('tileload', () => {
  setTimeout(() => {
    setNetworkError(prev => ({ ...prev, visible: false }))
  }, 500)
})

// 方式 3: 所有瓦片加载完成（兜底）
tileLayer.on('load', () => {
  setNetworkError(prev => ({ ...prev, visible: false }))
})
```

## 技术亮点

### 1. 零延迟错误检测

使用 `hasShownError` 标志避免重复检测，第一个瓦片失败就立即触发提示。

### 2. 三重保障恢复机制

1. **系统级**：`window.addEventListener('online')` - 最快
2. **瓦片级**：`tileload` 事件 - 次快
3. **完整级**：`load` 事件 - 兜底

### 3. 智能资源清理

使用 `useRef` 保存事件监听器引用，确保组件卸载时正确清理。

### 4. 用户友好

- 无需手动刷新
- 无需点击重试（除非想立即重试）
- 自动感知网络状态变化
