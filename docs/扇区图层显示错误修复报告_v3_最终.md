# LTE/NR 扇区图层显示错误修复报告 (v3 最终版)

**修复日期**: 2026-01-14
**问题优先级**: P0 (关键功能缺陷)
**影响范围**: LTE/NR 扇区图层切换功能

---

## 一、问题描述

### 症状

在图层控制面板点击显示 LTE 或 NR 扇区图层时,控制台报错,图层无法正常显示。

### 错误表现

- **初始加载**: 应用启动后自动加载图层时报错
- **手动切换**: 在图层控制面板勾选 LTE 或 NR 扇区图层复选框时报错
- **现象**: 控制台输出 `Cannot read properties of undefined` 错误信息
- **影响**: 扇区图层无法显示在地图上

---

## 二、根本原因分析

### 问题演进

#### 第一轮分析:图层初始化时机

**问题**: 图层仅在"可见"时才初始化,导致首次"隐藏"时图层永远不会被初始化。

**第一次修复**: 确保 `onAdd()` 始终被调用:
```typescript
// 先添加到地图（这会触发 onAdd 初始化）
lteSectorLayerRef.current.addTo(mapInstanceRef.current)
// 如果初始不可见,则移除
if (!layerVisibility.lte) {
  mapInstanceRef.current.removeLayer(lteSectorLayerRef.current)
}
```

#### 第二轮分析:useEffect 依赖冲突

**问题**: `createSectorLayers` 和可见性切换 useEffect 都依赖 `layerVisibility`,导致同时触发冲突。

**第二次修复**: 移除 `createSectorLayers` 的 `layerVisibility` 依赖,并添加 `hasLayer()` 检查:
```typescript
// 移除 layerVisibility 依赖
}, [sectorData, labelVisibility, handleSectorClick])

// 添加 hasLayer() 检查
const hasLayer = mapInstanceRef.current.hasLayer(lteSectorLayerRef.current)
if (layerVisibility.lte && !hasLayer) {
  lteSectorLayerRef.current.addTo(mapInstanceRef.current)
}
```

#### 第三轮分析:图层初始化时序问题

**问题**: 可见性切换 useEffect 在图层完全初始化之前就触发了,导致操作未初始化的图层。

**根本原因**:
```
应用启动流程:
┌─────────────────────────────────────┐
│ 1. 地图初始化完成                    │
├─────────────────────────────────────┤
│ 2. loadSectorData() 执行             │
├─────────────────────────────────────┤
│ 3. sectorData 更新                   │
│    ↓                                 │
│ 4. createSectorLayers() 触发         │
│    - 创建图层                         │
│    - addTo(map)                      │
│    - onAdd() 触发                    │
│      - featureGroup 初始化中...      │
│    - ⚠️ 还未完全初始化完成             │
├─────────────────────────────────────┤
│ 5. useEffect (L1407) 因为 layerVisibility 初始值触发 │
│    - 尝试 hasLayer() 检查             │
│    - 尝试 addTo() 或 removeLayer()    │
│    - ❌ 图层还未完全初始化,报错!       │
└─────────────────────────────────────┘
```

**时序冲突**:
- `createSectorLayers()` 中调用 `addTo(map)` 会**同步**触发 `onAdd()`
- `onAdd()` 中创建 `featureGroup` 也是**同步**的
- 但是 **React 的状态更新和 useEffect 触发是批处理的**
- 这意味着在 `createSectorLayers()` 还在执行时,另一个 useEffect 可能已经开始执行

---

## 三、最终修复方案

### 核心思路

**确保图层完全初始化后再操作可见性**:

在可见性切换 useEffect 中添加 `featureGroup` 检查,确保图层已经完全初始化。

### 修复后的代码

**文件**: [frontend/src/renderer/components/Map/OnlineMap.tsx:1407-1433](frontend/src/renderer/components/Map/OnlineMap.tsx#L1407-L1433)

```typescript
/**
 * 当图层可见性变化时更新
 */
useEffect(() => {
  if (!mapInstanceRef.current) return

  // ✅ 确保图层已经创建并添加到地图（通过检查 featureGroup）
  if (lteSectorLayerRef.current && (lteSectorLayerRef.current as any).featureGroup) {
    const hasLayer = mapInstanceRef.current.hasLayer(lteSectorLayerRef.current)
    if (layerVisibility.lte && !hasLayer) {
      lteSectorLayerRef.current.addTo(mapInstanceRef.current)
    } else if (!layerVisibility.lte && hasLayer) {
      mapInstanceRef.current.removeLayer(lteSectorLayerRef.current)
    }
  }

  if (nrSectorLayerRef.current && (nrSectorLayerRef.current as any).featureGroup) {
    const hasLayer = mapInstanceRef.current.hasLayer(nrSectorLayerRef.current)
    if (layerVisibility.nr && !hasLayer) {
      nrSectorLayerRef.current.addTo(mapInstanceRef.current)
    } else if (!layerVisibility.nr && hasLayer) {
      mapInstanceRef.current.removeLayer(nrSectorLayerRef.current)
    }
  }
}, [layerVisibility])
```

### 关键修复点

1. **添加 `featureGroup` 检查**: `(layerRef.current as any).featureGroup`
   - 确保图层的 `onAdd()` 已经执行完成
   - 确保内部的 `featureGroup` 已经创建

2. **保留 `hasLayer()` 检查**: 避免重复添加或移除

3. **移除 `createSectorLayers` 的 `layerVisibility` 依赖**: 避免 useEffect 冲突

### 问题 1: 图层初始化时机
