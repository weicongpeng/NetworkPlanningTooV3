# LTE/NR 扇区图层显示错误修复报告

**修复日期**: 2026-01-13
**问题优先级**: P0 (关键功能缺陷)
**影响范围**: LTE/NR 扇区图层切换功能

---

## 一、问题描述

### 症状

在图层控制面板点击显示 LTE 或 NR 扇区图层时,控制台报错,图层无法正常显示。

### 错误表现

- **操作**: 在图层控制面板勾选 LTE 或 NR 扇区图层复选框
- **现象**: 控制台输出错误信息
- **影响**: 扇区图层无法显示在地图上

---

## 二、根本原因分析

### 问题代码位置

**文件**: [frontend/src/renderer/components/Map/OnlineMap.tsx](frontend/src/renderer/components/Map/OnlineMap.tsx)

**相关代码行**: [L769-770](frontend/src/renderer/components/Map/OnlineMap.tsx#L769-L770) 和 [L787-788](frontend/src/renderer/components/Map/OnlineMap.tsx#L787-L788)

### 错误代码

```typescript
// 创建LTE扇区图层
if (sectorData.lte.length > 0) {
  lteSectorLayerRef.current = createSectorLayer({
    sectors: sectorData.lte,
    onClick: handleSectorClick,
    zoom: currentZoom,
    showLabels: labelVisibility.lte
  })
  if (layerVisibility.lte) {
    lteSectorLayerRef.current.addTo(mapInstanceRef.current)  // ❌ 问题
  }
  // ...
}
```

### 根本原因

`SectorSVGLayer` 是一个继承自 `L.Layer` 的自定义 Leaflet 图层类,其初始化流程如下:

1. **构造函数**: 仅设置属性,不创建 DOM
2. **`addTo(map)`**: 调用 Leaflet 的 `addTo()`,内部触发 `onAdd()` 方法
3. **`onAdd(map)`**:
   - 创建 `FeatureGroup` 管理器
   - 初始化事件监听
   - 执行首次渲染

**问题所在**: 原代码逻辑是"如果可见才添加到地图",这导致:
- 当 `layerVisibility.lte = false` 时,图层从未调用 `addTo(map)`
- `onAdd()` 永远不会被调用
- `featureGroup` 永远不会被初始化
- 后续即使切换为 `true`,由于 `featureGroup` 为 `undefined`,任何操作都会报错

### 时序问题

```
初始加载流程:
┌─────────────────────────────────────┐
│ 1. createSectorLayer()              │ → 创建图层实例
│    - mapInstance = undefined        │
│    - featureGroup = undefined       │
├─────────────────────────────────────┤
│ 2. if (layerVisibility.lte)         │ → 如果可见
│    - addTo(map)                      │   ├─ true  → ✅ 正常初始化
│    - 触发 onAdd()                    │   └─ false → ❌ 从未初始化
│      - featureGroup = L.featureGroup()│
├─────────────────────────────────────┤
│ 3. 用户点击复选框切换可见性          │
│    - setLayerVisibility(true)        │
│    - addTo(map)  ⚠️                 │ → 尝试添加
│      - featureGroup 未定义 → 报错!   │
└─────────────────────────────────────┘
```

## 三、修复方案

### 核心思路

**始终先初始化图层,然后根据初始可见性决定是否显示**:

```typescript
// 创建图层
lteSectorLayerRef.current = createSectorLayer({...})

// 始终先添加到地图（触发初始化）
lteSectorLayerRef.current.addTo(mapInstanceRef.current)

// 如果初始不可见,则立即移除
if (!layerVisibility.lte) {
  mapInstanceRef.current.removeLayer(lteSectorLayerRef.current)
}
```

## 六、总结

### 问题核心

**延迟初始化导致的状态不一致**: 图层仅在"可见"时才初始化,导致首次"隐藏"时图层永远不会被初始化。

### 修复方法

**强制初始化 + 条件显示**:
1. 创建图层后立即添加到地图(确保初始化)
2. 根据初始可见性决定是否立即移除
3. 后续切换只需正常的 `addTo()` / `removeLayer()`

### 修改影响

- **影响文件**: 1 个 ([OnlineMap.tsx](frontend/src/renderer/components/Map/OnlineMap.tsx))
- **影响行数**: ~15 行
- **破坏性**: 无
- **测试状态**: ✅ 构建通过

### 相关优化

此修复与之前的"扇区渲染虚拟化优化"兼容,两者协同工作:
- **渲染虚拟化**: 优化性能,减少 DOM 操作
- **初始化修复**: 确保图层正确加载
