## 一、资源消耗排名 (从高到低)

### 🔴 高消耗模块

#### 1. **地图渲染模块** (OnlineMap + SectorRendererSVG)
**位置**: `frontend/src/renderer/components/Map/OnlineMap.tsx`, `SectorRendererSVG.tsx`

**资源消耗**:
- **CPU**: ⭐⭐⭐⭐⭐ (极高)
- **内存**: ⭐⭐⭐⭐ (高)
- **GPU**: ⭐⭐⭐ (中等)

**消耗原因**:
```typescript
// SectorRendererSVG.tsx:326-363

  this.featureGroup.clearLayers()
  this.sectorPolygons.clear()

    // 创建 L.Polygon 或 L.CircleMarker
    // 每个扇区 = 1个 DOM 节点
  }
}
```

**关键性能瓶颈**:
1. **无缓存机制**: 每次缩放/移动都销毁并重建所有 DOM 节点
2. **全量渲染**: 即使视口外扇区也被渲染 (虽然有边界检查,但优化不足)
3. **高频触发**: `map.on('zoom')` 在缩放过程中持续触发
4. **requestAnimationFrame 节流不足**: 只防抖,未做真正优化
5. **无虚拟化**: 大量扇区时 DOM 节点过多 (5000+ 扇区 = 5000+ SVG 节点)

**实测数据** (推测):
- 5000 个扇区: 首次渲染 ~800ms, 缩放重渲染 ~200-400ms
- 10000 个扇区: 首次渲染 ~1500ms, 缩放重渲染 ~500-800ms
- 内存占用: 每个扇区约 5-10KB (包括 DOM + 数据)

## 二、详细性能优化建议

### 🎯 优先级 P0 (立即优化)

#### 优化 1: 实现扇区渲染虚拟化与缓存

**问题**: 当前每次缩放/移动都销毁并重建所有 DOM 节点

**解决方案**:
```typescript
// 在 SectorRendererSVG.tsx 中添加:

class SectorSVGLayer extends L.Layer {
  // 新增: 渲染缓存
  private renderedPolygons = new Map<string, {
    polygon: L.Polygon
    lastUsed: number  // 时间戳
  }>()

  private renderedMarkers = new Map<string, {
    marker: L.CircleMarker
    lastUsed: number
  }>()

  // 新增: 视口扇区缓存 (避免重复计算)
  private visibleSectorsCache: {
    bounds: string
    sectors: RenderSectorData[]
    timestamp: number
  } | null = null

  /**
   * 优化的渲染方法 - 仅更新变化的扇区
   */
  private _renderOptimized(): void {
    if (!this.mapInstance || !this.featureGroup) return

    const bounds = this.mapInstance.getBounds()
    const boundsKey = `${bounds.toBBoxString()}-${this.currentZoom}`

    // 检查缓存
    if (this.visibleSectorsCache?.bounds === boundsKey &&
        Date.now() - this.visibleSectorsCache.timestamp < 100) {
      return  // 100ms 内相同视口跳过
    }

    // 获取当前可见扇区
    const visibleSectors = this._getVisibleSectors(bounds)

    // 标记所有已渲染对象为"待删除"
    const currentRenderKeys = new Set<string>()

    // 添加或保留扇区
    for (const sector of visibleSectors) {
      const key = `${sector.sectorId || sector.cellId}`

      currentRenderKeys.add(key)

      if (!this.renderedPolygons.has(key)) {
        // 新扇区 - 创建并添加
        const polygon = this._createSectorPolygon(sector)
        polygon.addTo(this.featureGroup)
        this.renderedPolygons.set(key, {
          polygon,
          lastUsed: Date.now()
        })
      } else {
        // 已存在 - 更新时间戳
        const cached = this.renderedPolygons.get(key)!
        cached.lastUsed = Date.now()
      }
    }

    // 移除视口外的扇区 (延迟清理)
    const now = Date.now()
    for (const [key, { polygon, lastUsed }] of this.renderedPolygons) {
      if (!currentRenderKeys.has(key) && now - lastUsed > 2000) {
        // 超过 2 秒未使用才移除 (避免快速移动时频繁创建/销毁)
        this.featureGroup.removeLayer(polygon)
        this.renderedPolygons.delete(key)
      }
    }

    // 更新缓存
    this.visibleSectorsCache = {
      bounds: boundsKey,
      sectors: visibleSectors,
      timestamp: now
    }
  }

  /**
   * 优化的缩放处理 - 使用 requestAnimationFrame + 节流
   */
  private _onZoom(): void {
    if (this._animRequestId !== null) return

    this._animRequestId = L.Util.requestAnimFrame(() => {
      this._renderOptimized()
      this._animRequestId = null
    })
  }
}
```

**预期收益**:
- 减少 70-80% 的 DOM 操作
- 缩放重渲染从 200-400ms 降至 50-100ms
- 内存占用减少 30-40%

#### 优化 2: 坐标转换结果缓存

**问题**: 每个要素的坐标都重复转换,相同坐标转换多次

**解决方案**:
```typescript
// 在 coordinate.ts 中添加:

class CoordinateTransformer {
  // 转换缓存 (精度 6 位小数 = ~0.1 米)
  private static cache = new Map<string, { lat: number; lng: number }>()
  private static MAX_CACHE_SIZE = 10000

  static wgs84ToGcj02(lat: number, lng: number): [number, number] {
    // 缓存键 (精度 6 位)
    const key = `${lat.toFixed(6)},${lng.toFixed(6)}`

    if (this.cache.has(key)) {
      return [this.cache.get(key)!.lat, this.cache.get(key)!.lng]
    }

    // 执行转换
    const result = super.wgs84ToGcj02(lat, lng)

    // 缓存结果
    if (this.cache.size >= this.MAX_CACHE_SIZE) {
      // LRU: 删除最早的 10%
      const entries = Array.from(this.cache.entries())
      entries.slice(0, 1000).forEach(([k]) => this.cache.delete(k))
    }

    this.cache.set(key, { lat: result[0], lng: result[1] })

    return result
  }
}
```

**预期收益**:
- 坐标转换速度提升 80-90%
- MapInfo 图层加载时间减少 40-50%

class MapInfoDataManager {
  private loadedTiles = new Map<string, GeoJSON.FeatureCollection>()

  async loadLayerTile(
    layerId: string,
    bounds: L.LatLngBounds,
    zoom: number
  ): Promise<GeoJSON.FeatureCollection> {
    // 计算瓦片键
    const tileKey = this._getTileKey(bounds, zoom)

    if (this.loadedTiles.has(tileKey)) {
      return this.loadedTiles.get(tileKey)!
    }

    // 请求瓦片数据

    this.loadedTiles.set(tileKey, data)

    return data
  }

  cleanupVisibleTiles(visibleTiles: Set<string>) {
    for (const key of this.loadedTiles.keys()) {
      if (!visibleTiles.has(key)) {
        this.loadedTiles.delete(key)
      }
    }
  }
}

private _renderLabels(): void {

  if (this.currentZoom < 15) {
    // 清除所有标签
    this.sectorLabels.forEach(label => this.mapInstance?.removeLayer(label))

  }
}

建议在关键位置添加性能监控:

// performance-monitor.ts

export class PerformanceMonitor {
  static measureRender(componentName: string, fn: () => void) {
    const start = performance.now()
    fn()
    const duration = performance.now() - start

    // 上报到监控服务
    if (duration > 100) {
      console.warn(`[Performance] ${componentName} slow render: ${duration.toFixed(2)}ms`)
    }

    return duration
  }

  static measureMemory() {
    if ('memory' in performance) {
      return (performance as any).memory.usedJSHeapSize / 1024 / 1024
    }
    return 0
  }
}

## 五、总结

### 当前主要瓶颈
1. **地图渲染**: 全量 DOM 操作,无缓存
2. **坐标转换**: 重复计算,无缓存

引入 WebGL、Web Worker 等技术,实现大规模数据支持。
