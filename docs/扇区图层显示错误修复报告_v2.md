# LTE/NR 扇区图层显示错误修复报告 (v2)

**修复日期**: 2026-01-14
**问题优先级**: P0 (关键功能缺陷)
**影响范围**: LTE/NR 扇区图层切换功能

---

## 一、问题描述

### 症状

在图层控制面板点击显示 LTE 或 NR 扇区图层时,控制台报错,图层无法正常显示。

### 错误表现

- **操作**: 在图层控制面板勾选 LTE 或 NR 扇区图层复选框
- **现象**: 控制台输出错误信息
- **影响**: 扇区图层无法显示在地图上

---

## 二、根本原因分析

### 第一次修复尝试

**文件**: [frontend/src/renderer/components/Map/OnlineMap.tsx](frontend/src/renderer/components/Map/OnlineMap.tsx)

**相关代码行**: L762-824 (`createSectorLayers` 函数)

**第一次修复思路**:
```typescript
// 创建LTE扇区图层
if (sectorData.lte.length > 0) {
  lteSectorLayerRef.current = createSectorLayer({...})
  // 先添加到地图（这会触发 onAdd 初始化）
  lteSectorLayerRef.current.addTo(mapInstanceRef.current)

  // 如果初始不可见,则移除
  if (!layerVisibility.lte) {
    mapInstanceRef.current.removeLayer(lteSectorLayerRef.current)
  }
}
```

**第一次修复的问题**:
虽然确保了图层的 `onAdd()` 被调用(初始化 `featureGroup`),但引入了新的问题:**useEffect 依赖冲突**。

### 真正的根本原因

**useEffect 依赖冲突**:

1. **L1293-1297**: 监听 `sectorData` 和 `createSectorLayers` 变化
   ```typescript
   useEffect(() => {
     if (mapInstanceRef.current && sectorData) {
       createSectorLayers()
     }
   }, [sectorData, createSectorLayers])
   ```

2. **L1370-1388**: 监听 `layerVisibility` 变化
   ```typescript
   useEffect(() => {
     if (lteSectorLayerRef.current) {
       if (layerVisibility.lte) {
         lteSectorLayerRef.current.addTo(mapInstanceRef.current)
       } else {
         mapInstanceRef.current.removeLayer(lteSectorLayerRef.current)
       }
     }
   }, [layerVisibility])
   ```

3. **`createSectorLayers` 的依赖项** (L804):
   ```typescript
   }, [sectorData, layerVisibility, labelVisibility, handleSectorClick])
   ```

**问题流程**:

```
用户点击复选框
    ↓
setLayerVisibility() 被调用
    ↓
layerVisibility state 更新
    ↓
同时触发两个 useEffect:
    ├─ useEffect (L1293) 因为 createSectorLayers 依赖变化
    │   ↓
    │   createSectorLayers() 被调用
    │   ↓
    │   创建新图层实例
    │   ↓
    │   调用 addTo(map)
    │   ↓
    │   如果初始不可见则 removeLayer()
    │
    └─ useEffect (L1370) 因为 layerVisibility 直接变化
        ↓
        也调用 addTo(map) 或 removeLayer()
        ↓
        与上面的操作冲突!
```

**具体冲突场景**:

1. **初始加载** (`layerVisibility = { lte: false, nr: false }`):
   - `createSectorLayers()` 创建图层 → `addTo(map)` → `onAdd()` 初始化 `featureGroup`
   - 然后立即 `removeLayer()` → `onRemove()` → `featureGroup = undefined`
   - **同时** L1370 的 useEffect 也触发,尝试 `removeLayer()` (已经不在地图上,可能报错)

2. **用户切换可见性** (`layerVisibility.lte` 从 `false` → `true`):
   - `createSectorLayers()` 重新创建图层 → `addTo(map)`
   - **同时** L1370 的 useEffect 也触发,再次 `addTo(map)`
   - **重复添加导致错误**

## 三、修复方案

### 核心思路

**职责分离**:

1. **`createSectorLayers`**: 只负责创建图层和初始化,不处理可见性
2. **`useEffect (L1370)`**: 专门负责图层可见性的切换

### 修复后的代码

**修改 1**: 移除 `createSectorLayers` 的 `layerVisibility` 依赖

**文件**: [frontend/src/renderer/components/Map/OnlineMap.tsx:746-804](frontend/src/renderer/components/Map/OnlineMap.tsx#L746-L804)

```typescript
/**
 * 创建扇区图层(不处理可见性)
 */
const createSectorLayers = useCallback(() => {
  if (!mapInstanceRef.current) return

  const currentZoom = mapInstanceRef.current.getZoom()

  // 移除旧图层
  if (lteSectorLayerRef.current) {
    mapInstanceRef.current.removeLayer(lteSectorLayerRef.current)
    lteSectorLayerRef.current = null
  }
  if (nrSectorLayerRef.current) {
    mapInstanceRef.current.removeLayer(nrSectorLayerRef.current)
    nrSectorLayerRef.current = null
  }

  // 创建LTE扇区图层
  if (sectorData.lte.length > 0) {
    lteSectorLayerRef.current = createSectorLayer({
      sectors: sectorData.lte,
      onClick: handleSectorClick,
      zoom: currentZoom,
      showLabels: labelVisibility.lte
    })
    // 先添加到地图（这会触发 onAdd 初始化）
    lteSectorLayerRef.current.addTo(mapInstanceRef.current)

    // 应用当前频点可见性
    const visibilityMap = new Map<number, boolean>()
    frequencies.lte.forEach(f => visibilityMap.set(f.frequency, f.visible))
    lteSectorLayerRef.current.setFrequenciesVisibility(visibilityMap)
  }

  // 创建NR扇区图层
  if (sectorData.nr.length > 0) {
    nrSectorLayerRef.current = createSectorLayer({
      sectors: sectorData.nr,
      onClick: handleSectorClick,
      zoom: currentZoom,
      showLabels: labelVisibility.nr
    })
    // 先添加到地图（这会触发 onAdd 初始化）
    nrSectorLayerRef.current.addTo(mapInstanceRef.current)

    // 应用当前频点可见性
    const visibilityMap = new Map<number, boolean>()
    frequencies.nr.forEach(f => visibilityMap.set(f.frequency, f.visible))
    nrSectorLayerRef.current.setFrequenciesVisibility(visibilityMap)
  }
  // ✅ 移除了 layerVisibility 依赖
}, [sectorData, labelVisibility, handleSectorClick])
```

**修改 2**: 增强 L1370-1388 useEffect 的安全检查

**文件**: [frontend/src/renderer/components/Map/OnlineMap.tsx:1367-1393](frontend/src/renderer/components/Map/OnlineMap.tsx#L1367-L1393)

```typescript
/**
 * 当图层可见性变化时更新
 */
useEffect(() => {
  if (!mapInstanceRef.current) return

  if (lteSectorLayerRef.current) {
    // ✅ 添加安全检查:避免重复添加或移除
    const hasLayer = mapInstanceRef.current.hasLayer(lteSectorLayerRef.current)
    if (layerVisibility.lte && !hasLayer) {
      lteSectorLayerRef.current.addTo(mapInstanceRef.current)
    } else if (!layerVisibility.lte && hasLayer) {
      mapInstanceRef.current.removeLayer(lteSectorLayerRef.current)
    }
  }

  if (nrSectorLayerRef.current) {
    // ✅ 添加安全检查:避免重复添加或移除
    const hasLayer = mapInstanceRef.current.hasLayer(nrSectorLayerRef.current)
    if (layerVisibility.nr && !hasLayer) {
      nrSectorLayerRef.current.addTo(mapInstanceRef.current)
    } else if (!layerVisibility.nr && hasLayer) {
      mapInstanceRef.current.removeLayer(nrSectorLayerRef.current)
    }
  }
}, [layerVisibility])
```

## 五、关键修复点总结

### 问题 1: useEffect 依赖冲突

**原因**: `createSectorLayers` 和可见性切换 useEffect 都依赖 `layerVisibility`,导致同时触发。

**修复**: 移除 `createSectorLayers` 的 `layerVisibility` 依赖,让它只负责创建图层。

### 问题 2: 重复添加/移除图层

**原因**: 两个 useEffect 都调用 `addTo()` 或 `removeLayer()`,可能重复操作。

**修复**: 在可见性切换 useEffect 中添加 `hasLayer()` 检查,避免重复操作。

### 问题 3: 图层初始化时机

**原因**: 如果图层在不可见状态下创建,可能不会被正确初始化。

**修复**: `createSectorLayers` 始终调用 `addTo(map)` 确保初始化,然后由可见性 useEffect 控制显示/隐藏。
