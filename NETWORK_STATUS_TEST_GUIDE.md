# 网络异常提示测试指南

## 测试场景

### 1. 完全断网测试

**步骤**：
1. 打开网络规划工具应用
2. 进入地图页面
3. 断开网络连接（关闭 WiFi 或拔掉网线）
4. 刷新页面或重新打开地图

**预期结果**：
- 地图容器显示空白或部分灰色
- **右下角或底部出现红色警告框**：
  - 显示 "网络连接异常" 图标（WifiOff）
  - 提示文字："在线地图瓦片加载失败，请检查网络连接"
  - 有 "重试" 按钮
  - 有关闭（X）按钮
- 如果右上角有网络状态指示器，应显示红色离线状态

**如果看不到提示**：
- 检查浏览器控制台是否有 `[OnlineMap] 瓦片加载失败` 日志
- 检查是否有 JavaScript 错误
- 等待最多 8 秒（超时检测）

### 2. 后端服务关闭测试

**步骤**：
1. 保持网络连接
2. 关闭后端服务（`python main.py`）
3. 刷新地图页面

**预期结果**：
- 地图可能正常显示（使用高德在线瓦片）
- 但无法加载工参数据
- 如果尝试加载数据，会显示错误提示

### 3. 网络恢复测试

**步骤**：
1. 在断网状态下，地图显示网络错误提示
2. 重新连接网络
3. 点击 "重试" 按钮

**预期结果**：
- 网络恢复正常后，错误提示应自动消失
- 地图瓦片开始正常加载

### 4. 慢速网络测试

**步骤**：
1. 打开 Chrome DevTools（F12）
2. 切换到 Network 面板
3. 在 Throttling 下拉菜单中选择 "Slow 3G"
4. 刷新地图页面

**预期结果**：
- 地图加载缓慢
- 如果超过 8 秒仍未加载完成，显示超时提示

## 浏览器 DevTools 测试方法

### 方法 1: 离线模式

1. 打开 DevTools（F12）
2. 切换到 Network 面板
3. 勾选 "Offline" 复选框
4. 刷新页面

### 方法 2: 阻止特定请求

1. 打开 DevTools（F12）
2. 切换到 Network 面板
3. 打开 "Network request blocking" 设置
4. 添加模式：`*/_wm.appmaptile*` 或 `*webrd02.is.autonavi.com*`

### 方法 3: 模拟服务器错误

1. 打开 DevTools（F12）
2. 切换到 Network 面板
3. 右键点击 → 选择 "Online" → 选择 "Offline"

## 验证检查点

### ✅ 视觉检查

- [ ] 错误提示框出现（固定在底部中央）
- [ ] 提示框有雷达扫描动画效果（红色旋转扫描线）
- [ ] 图标正确（WifiOff 图标）
- [ ] 文字清晰可读
- [ ] 重试按钮可点击
- [ ] 关闭按钮可点击

### ✅ 功能检查

- [ ] 点击重试按钮触发重新加载
- [ ] 点击关闭按钮隐藏提示
- [ ] 网络恢复后提示自动消失
- [ ] 控制台有相关日志输出

在浏览器控制台应看到以下日志：

```

[OnlineMap] 检测到多个瓦片加载失败，可能是网络问题
```

## 常见问题排查

### Q: 断网后没有任何提示？

**A:** 可能的原因：
1. 页面已缓存，需要强制刷新（Ctrl+Shift+R）
2. JavaScript 执行错误，检查控制台
3. 代码未正确部署，检查文件是否保存

### Q: 提示显示但样式错误？

**A:** 检查：
1. Tailwind CSS 是否正确加载
2. `index.css` 中的动画定义是否存在
3. 浏览器是否支持 CSS 动画

### Q: 点击重试没有反应？

**A:** 检查：
1. `retryInitMap` 函数是否正确定义
2. 浏览器控制台是否有错误
3. 页面是否允许重载（某些嵌入式环境可能限制）

## 性能验证

### 瓦片加载性能

正常网络下：
- 首屏瓦片应在 2-3 秒内加载完成
- 不应显示错误提示

慢速网络下：
- 可能有延迟
- 但最终应加载成功

### 内存泄漏检查

1. 打开 DevTools → Memory 面板
2. 拍摄堆快照
3. 断网 → 重试多次
4. 再次拍摄快照
5. 对比两次快照，检查是否有内存泄漏
