# 网络异常提示修复验证指南

## 快速验证步骤

### 测试 1：正常网络（不应显示提示）

**步骤**：
1. 确保网络连接正常
2. 打开地图页面
3. 等待地图完全加载

**预期结果**：
- ✅ 地图正常显示
- ✅ **没有任何错误提示**
- ✅ 控制台显示：`[OnlineMap] 瓦片加载成功`

### 测试 2：模拟网络断开（应显示提示）

**步骤**：
1. 打开地图页面
2. 打开浏览器 DevTools（F12）
3. 切换到 Network 面板
4. 勾选 "Offline" 复选框
5. 刷新页面

**预期结果**：
- ✅ **立即显示**网络错误提示（< 0.5 秒）
- ✅ 提示框显示在底部中央
- ✅ 有重试和关闭按钮
- ✅ 控制台显示：`[OnlineMap] 检测到初始瓦片加载失败，可能是网络问题`

### 测试 3：网络恢复（应自动清除）

**步骤**：
1. 在断网状态下（已显示错误提示）
2. 取消 "Offline" 勾选
3. **不要手动刷新页面**

**预期结果**：
- ✅ **< 1 秒内**提示自动消失
- ✅ 地图开始正常加载
- ✅ 控制台显示：`[OnlineMap] 浏览器检测到网络恢复`

### 测试 4：点击重试（应重新加载）

**步骤**：
1. 在断网状态下
2. 点击错误提示中的 "重试" 按钮

**预期结果**：
- ✅ 页面自动刷新
- ✅ 重新尝试加载地图
- ✅ 如果网络恢复正常，加载成功

### 测试 5：切换地图类型（不应误报）

**步骤**：
1. 在网络正常状态下
2. 多次点击地图类型切换按钮（平面/卫星）
3. 观察是否有错误提示

**预期结果**：
- ✅ 地图正常切换
- ✅ **没有任何错误提示**
- ✅ 控制台显示：`[OnlineMap] 切换地图类型后瓦片加载成功`

## 常见问题排查

### Q1: 网络正常但还是显示错误提示？

**检查**：
1. 打开控制台，查看是否有 `[OnlineMap] 已有瓦片成功加载，忽略个别瓦片加载失败`
2. 如果没有这条日志，说明检测逻辑有问题

**可能原因**：
- 代码未正确部署
- 浏览器缓存了旧代码

### Q2: 断网后没有提示？

**检查**：
1. 确保使用 DevTools 的 "Offline" 模式
2. 检查控制台是否有 `tileerror` 事件日志
3. 等待至少 3 秒（超时检测）

### Q3: 网络恢复后提示不消失？

**检查**：
1. 查看控制台是否有 `online` 事件日志
2. 尝试手动点击关闭按钮
3. 刷新页面看是否恢复正常

### Q4: 切换地图类型时显示错误？

**检查**：
1. 确保网络连接正常
2. 查看控制台是否有切换相关日志
3. 如果持续出现问题，可能是高德地图服务问题

## 调试命令

在浏览器控制台运行以下命令进行调试：

```javascript
// 检查网络状态
console.log('Online:', navigator.onLine)

// 检查地图实例
const map = window.L?.map?.getContainer()?.parentNode
console.log('Map exists:', !!map)

// 检查瓦片图层
const mapInstance = document.querySelector('.leaflet-container')
console.log('Map container exists:', !!mapInstance)

// 监听网络状态变化
window.addEventListener('online', () => console.log('Network is online'))
window.addEventListener('offline', () => console.log('Network is offline'))
```

## 性能监控

### 网络请求监控

1. 打开 DevTools → Network 面板
2. 筛选 "Tile" 或 "Map" 请求
3. 观察瓦片请求的状态

**正常情况**：
- 大部分瓦片请求状态码为 200
- 偶尔有 404 或失败是正常的

**网络异常**：
- 大部分瓦片请求失败
- 请求状态为 (failed) net::ERR_INTERNET_DISCONNECTED

### 内存监控

1. 打开 DevTools → Memory 面板
2. 多次触发网络异常和恢复
3. 拍摄堆快照对比

**预期**：
- 没有明显的内存增长
- 定时器被正确清理
